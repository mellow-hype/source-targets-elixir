Aide Outils de diagnostic

Les outils de diagnostic (PING et Traceroute) vous permettent de vérifier les connexions de vos composants réseau.

Outil de diagnostic - Cliquez sur le bouton radio pour sélectionner un outil de diagnostic :

  • PING - Cet outil de diagnostic permet de résoudre les problèmes de connectivité, d'accessibilité et de résolution de nom consistant à envoyer une requête ICMP Echo et à attendre une réponse Echo reply d'un hôte ou d'une passerelle. Vous pouvez utiliser le PING pour tester à la fois une adresse IP numérique et un nom de domaine. Si le PING de l'adresse IP est concluant, mais que celui du nom de domaine échoue, vous avez certainement un problème de résolution de nom. Dans ce cas, essayez de résoudre le problème de nom de domaine à l'aide des requêtes DNS.
  • Traceroute - Cet utilitaire détermine le chemin d'un paquet de données jusqu'à un hôte donné en envoyant des messages ICMP (Echo Request) avec des valeurs TTL variables jusqu'à destination. Au cours du transfert, chaque passerelle du chemin décrémente d'au moins 1 la valeur TTL du paquet IP. Le TTL indique le nombre maximal de liaisons. Lorsque le TTL sur un paquet est de 0, la passerelle envoie un message d'erreur ICMP Time Exceeded à votre périphérique. Traceroute détermine le chemin en envoyant un premier message Echo Request avec un TTL égal à 1, puis l'incrémente de 1 à chaque transmission jusqu'à ce que la cible réponde ou que le maximum de sauts soit atteint. Le nombre maximum par défaut est de 20. Il peut être spécifié dans « TTL max traceroute ». Le chemin est déterminé par l'examen des messages ICMP Time Exceeded envoyés par les passerelles intermédiaires, ainsi que par le message Echo Reply renvoyé. Cependant, certaines passerelles n'envoient pas de messages Time Exceeded pour les paquets dont le TTL a expiré et sont donc invisibles pour l'outil traceroute. Dans ce cas, le traceroute affiche une ligne d'astérisques (*) pour ce saut.

Adresse IP/nom domaine - Saisissez l'adresse IP ou le nom de domaine du PC dont vous souhaitez analyser la connexion.

Nombre de PING - Indique le nombre de messages Echo Request envoyés. La valeur par défaut est 4.

Taille du paquet PING - Indique le nombre d'octets à transmettre. La valeur par défaut est 64.

Délai d'attente dépassé - Temps d'attente pour une réponse, en millisecondes. La valeur par défaut est 800.

TTL max. traceroute - Paramétrez le nombre maximal de sauts (TTL max. à atteindre) sur le chemin pour chercher la cible (destination). La valeur par défaut est 20.

Cliquez sur Démarrer pour lancer la procédure de diagnostic.

Remarque :

  1. Un seul utilisateur à la fois peut utiliser les outils de diagnostic.
  2. « Nombre de PING », « Taille du paquet PING » et « Délai d'attente dépassé » sont des Paramètres PING. « Traceroute Max TTL » est un Paramètre Traceroute.
