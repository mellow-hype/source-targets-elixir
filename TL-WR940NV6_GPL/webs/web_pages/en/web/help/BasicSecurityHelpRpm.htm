Aide Sécurité de base

Sur cette page, vous pouvez configurer les Paramètres de sécurité de base.

Pare-feu - Permet de (dés)activer le pare-feu du routeur.

  • Pare-feu SPI - La fonction d'inspection dynamique des paquets (SPI) permet de prévenir les cyber-attaques en contrôlant les paquets. Elle vérifie que le trafic de la session est conforme au protocole. Le pare-feu SPI est activé par défaut. Si vous désactivez cette fonction, tous les ordinateurs du réseau local seront exposés aux attaques extérieures.

VPN - Le Passthrough VPN doit être activé pour que les tunnels VPN puissent utiliser les protocoles VPN pour traverser le routeur.

  • Passthrough PPTP - Passthrough PPTP. Le protocole de tunnellisation point à point (PPTP) permet l'encapsulation du protocole point à point (PPP) par réseau IP. Pour autoriser les tunnels PPTP à traverser le routeur, cliquez sur Activer.
  • Passthrough L2TP - Le protocole de tunnellisation de niveau 2 (L2TP) permet d'activer des sessions point à point via Internet sur la couche 2. Pour autoriser les tunnels L2TP à traverser le routeur, cliquez sur Activer.
  • Passthrough IPSec - Le protocole de sécurité IP IPSec (Internet Protocol security) est une suite de protocoles assurant des communications privées et protégées sur des réseaux IP, grâce à l'utilisation de services de sécurité cryptographiques. Pour autoriser les tunnels IPSec à traverser le routeur, cliquez sur Activer.

ALG - Il est conseillé d'activer le service de la passerelle de la couche Application (ALG) pour connecter des filtres de traversée NAT personnalisés sur la passerelle et prendre en charge la traduction d'adresses et de ports pour certains protocoles de contrôle/données de la couche application (FTP, TFTP, H323, etc.)

  • FTP ALG - Pour autoriser les clients et serveurs FTP à transférer des données via NAT, cliquez sur Activer.
  • TFTP ALG - Pour autoriser les clients et serveurs TFTP à transférer des données via NAT, cliquez sur Activer.
  • H323 ALG - Pour autoriser les clients NetMeeting Microsoft à communiquer via NAT, cliquez sur Activer.
  • RTSP ALG - Pour autoriser des clients Media Player à communiquer avec des serveurs multimédia en streaming via NAT, cliquez sur Activer.

Cliquez sur Enregistrer pour sauvegarder vos paramètres.
