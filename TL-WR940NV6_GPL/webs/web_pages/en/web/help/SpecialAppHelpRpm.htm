Aide Déclenchement de ports

Certaines applications, tels que les jeux en ligne, la visioconférence ou la téléphonie sur IP, nécessitent des connexions multiples. Le déclenchement de ports sert à certaines d'entre elles, qui ne peuvent fonctionner avec un routeur NAT simple.

Description du fonctionnement :

  1. Un hôte local se connecte à un hôte externe à l'aide du numéro de port de destination défini dans le champ Port de déclenchement.
  2. Le routeur enregistre cette connexion, ouvre le ou les ports entrants associés à cette entrée dans la table de Déclenchement de ports et les associe à l'hôte local.
  3. Si nécessaire, l'hôte externe peut se connecter à l'hôte local à l'aide d'un des ports définis dans le champ Ports entrants.

Règles :

  • Port de déclenchement - Port trafic sortant. Une connexion sortante sur ce port déclenchera la règle.
  • Protocole de déclenchement - Protocole utilisé pour les ports de déclenchement : TCP, UDP ou Tous (tous les protocoles pris en charge par le routeur).
  • Port entrant - Plage de ports ou port utilisé par le système distant pour répondre à la requête sortante. Une réponse utilisant un de ces ports est transférée à l'ordinateur qui a déclenché cette règle. Vous pouvez saisir jusqu'à 5 groupes de ports. Chaque groupe de ports doit être séparé par une virgule. Ex. : 2000-2038, 2046, 2050-2051, 2085, 3010-3030.
  • Protocole entrant - Protocole utilisé pour le port entrant : TCP, UDP ou Tous (tous les protocoles pris en charge par le routeur).
  • État - État de l'entrée : Activé signifie que le déclenchement de ports est activé.
  • Modifier - Permet de modifier ou de supprimer une entrée.
  • Applications communes - Applications déjà présentes dans la liste déroulante Protocole entrant.

Pour ajouter une nouvelle règle, procédez comme suit sur l'écran Déclenchement de ports :

  1. Cliquez sur le bouton Ajouter....
  2. Dans le champ Port de déclenchement, entrez le numéro de port utilisé par l'application pour envoyer une requête.
  3. Dans la liste déroulante Port de déclenchement, sélectionnez le protocole utilisé : TCP, UDP ou Tous.
  4. Dans le champ Ports entrants, saisissez la plage de ports utilisée par le système distant pour répondre à une requête.
  5. Dans la liste déroulante Ports entrants, sélectionnez le protocole utilisé : TCP, UDP ou Tous.
  6. Sélectionnez Activé dans la liste déroulante du champ État.
  7. Cliquez sur le bouton Enregistrer pour sauvegarder la nouvelle règle.

La liste Applications communes énumère de nombreuses applications. Sélectionnez-en-une ; les champs Port de déclenchement et Ports entrants seront automatiquement remplis. Il en ira de même si vous ajoutez une nouvelle règle.

Modifier ou supprimer une entrée :

  1. Recherchez l'entrée souhaitée dans la table.
  2. Cliquez sur Modifier ou Supprimer dans la colonne Modifier.

Cliquez sur le bouton Tout activer pour activer toutes les entrées.

Cliquez sur le bouton Tout désactiver pour désactiver toutes les entrées.

Cliquez sur le bouton Tout Supprimer pour effacer toutes les entrées.

Cliquez sur Suivant ou Précédent pour accéder à la page souhaitée.

Remarque :

  1. Dès que la connexion de déclenchement est établie, les ports ouverts correspondants sont fermés.
  2. Une règle ne peut être utilisée que par un seul hôte du réseau local à la fois. Toute autre connexion déclenchée par un autre hôte du LAN sera refusée.
  3. Les plages des Ports entrants ne peuvent se chevaucher.
